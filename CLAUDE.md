# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
npm start        # Launch the Electron app in development mode
npm run build    # Build Windows NSIS installer (x64)
npm run build:dir # Build portable .exe (unpacked, x64)
npm install      # Install dependencies
```

## Architecture

**Electron desktop app** with three layers:

- **`main.js`** — Electron main process. Creates BrowserWindow (1200x800, min 800x600), handles file dialogs (open/save/save-as), reads/writes files via `fs`, implements single-instance lock, and parses command-line args for file association (`--` flags and `.md`/`.markdown` extensions). IPC handlers: `open-file`, `save-file`, `save-file-as`, `get-file-path`.

- **`preload.js`** — Context bridge exposing `window.electronAPI` to the renderer. Methods: `openFile()`, `saveFile(content)`, `saveFileAs(content)`, `getFilePath()`, `onFileOpened(callback)`. Uses `contextIsolation: true` (no `nodeIntegration`).

- **`直接打开用.html` (index.html)** — The renderer UI. Contains everything: HTML structure, all CSS (Catppuccin-inspired dark theme via CSS custom properties), and vanilla JS logic. No frameworks or UI libraries. Key features:
  - Custom regex-based Markdown parser (no `marked`/`remark` dependency) supporting headers, bold, italic, strikethrough, code blocks, inline code, links, images, blockquotes, lists, tables, and horizontal rules
  - Two modes: reading (centered content at max-width 800px) and edit (50/50 horizontal split with left textarea + right live preview)
  - 150ms debounced preview updates during editing
  - Empty state ("Open a file to begin") shown when no file is loaded
  - Keyboard shortcuts: Ctrl+O (open), Ctrl+S (save), Ctrl+E (toggle mode)
  - Falls back to `<input type="file">` when File System Access API is unavailable

## Build Output

- **Installer**: `installer/Markdown Editor Setup 1.0.0.exe` (NSIS)
- **Portable**: Generated by `build:dir` — unpacked directory

Both builds register file associations for `.md` and `.markdown`.

## Notable Design Choices

- Markdown is parsed client-side with regex — no external parsing library. The parser is in `parseMarkdown()` and `renderTable()` functions.
- Security: `contextIsolation: true`, `nodeIntegration: false`. Renderer never touches Node.js APIs directly.
- The standalone HTML file (`直接打开用.html`) works independently in a browser — it uses the File System Access API instead of Electron IPC.
- Color palette defined as CSS custom properties in `:root` — all colors are derived from these variables.
