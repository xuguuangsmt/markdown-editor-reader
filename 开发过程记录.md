# Markdown Editor & Reader 开发过程记录

## 1. 用户需求

用户提出创建一个 Markdown 编辑和阅读软件，要求：
- 默认为阅读 Markdown 文件
- 编辑模式时，界面左边为 Markdown 格式编辑，右边为显示效果

后续扩展需求：生成 Windows 可执行程序 (exe)

---

## 2. 第一阶段：Web 版本实现

### 2.1 方案选择

**技术选型**：纯 HTML + CSS + JavaScript 单文件应用

**理由**：
- 无需构建工具，直接浏览器即可运行
- 轻量级，原型验证快
- 便于后续迁移到 Electron

### 2.2 实现要点

- **阅读模式**：居中显示渲染后的 Markdown 内容
- **编辑模式**：CSS Flexbox 实现左右分栏布局 (50/50)
- **Markdown 解析**：使用正则表达式实现基础解析器，支持标题、粗体、斜体、代码块、列表、表格等
- **实时预览**：150ms 防抖更新

---

## 3. 第二阶段：Windows 可执行程序

### 3.1 方案决策过程

#### 方案一：Tauri 框架

**初始选择理由**：
- Rust 后端，性能优异
- 生成的二进制文件小 (5-10MB)
- 安全沙箱机制

**遇到的问题**：
```
rustc --version  # 命令不可用
cargo --version  # 命令不可用
```

**尝试解决**：
```powershell
# 尝试安装 Rust
Invoke-WebRequest -Uri "https://win.rustup.rs" -OutFile "$env:TEMP\rustup-init.exe"
Start-Process -FilePath "$env:TEMP\rustup-init.exe" -ArgumentList "-y" -Wait
```

**结果**：Rust 安装过程中挂起或失败，系统未检测到 Rust 工具链

#### 方案二：Electron 框架

**最终选择理由**：
- 只需 Node.js 环境（系统已安装 v24.14.1）
- 生态成熟，文档完善
- 适合桌面应用开发

**决策依据**：
| 考量因素 | Tauri | Electron |
|---------|-------|----------|
| 依赖环境 | 需要 Rust | 需要 Node.js ✅ |
| 安装难度 | 需编译安装 | npm 一键安装 ✅ |
| 社区支持 | 较小众 | 成熟庞大 ✅ |
| 文件大小 | 5-10MB | 150-200MB |

### 3.3 Electron 实现细节

#### 项目结构
```
markdown/
├── main.js          # Electron 主进程
├── preload.js       # 安全桥接脚本
├── index.html       # 渲染进程界面
├── package.json     # 项目配置
└── dist/            # 构建输出
```

#### 核心技术点

**1. 主进程 (main.js)**
```javascript
const { app, BrowserWindow, ipcMain, dialog } = require('electron');
const fs = require('fs');

// 创建窗口
mainWindow = new BrowserWindow({
    width: 1200,
    height: 800,
    webPreferences: {
        nodeIntegration: false,
        contextIsolation: true,
        preload: path.join(__dirname, 'preload.js')
    }
});

// 文件对话框
ipcMain.handle('open-file', async () => {
    const result = await dialog.showOpenDialog({
        filters: [
            { name: 'Markdown Files', extensions: ['md', 'markdown', 'txt'] }
        ]
    });
    // 读取并返回文件内容
});

// 文件保存
ipcMain.handle('save-file', async (event, content) => {
    fs.writeFileSync(currentFilePath, content, 'utf-8');
});
```

**2. 预加载脚本 (preload.js)**
```javascript
const { contextBridge, ipcRenderer } = require('electron');

contextBridge.exposeInMainWorld('electronAPI', {
    openFile: () => ipcRenderer.invoke('open-file'),
    saveFile: (content) => ipcRenderer.invoke('save-file', content)
});
```

**3. 渲染进程 (index.html)**
- 界面与 Web 版本一致
- 通过 `window.electronAPI` 调用原生能力
- 支持拖拽标题栏移动窗口

**4. 构建配置 (package.json)**
```json
{
  "scripts": {
    "start": "electron .",
    "build": "electron-builder --win --x64",
    "build:dir": "electron-builder --win --x64 --dir"
  }
}
```

#### 构建命令
```bash
# 安装依赖（使用国内镜像）
ELECTRON_MIRROR="https://npmmirror.com/mirrors/electron/" npm install

# 打包为可执行文件
npm run build:dir
```

#### 构建输出
```
dist/win-unpacked/
└── markdown-editor-reader.exe  # 可直接运行的可执行文件
```

---

## 4. 总结

| 阶段 | 技术方案 | 结果 |
|------|---------|------|
| Web 原型 | HTML + JS 单文件 | 完成，提供浏览器预览 |
| Windows 应用 | Electron | 完成，生成 exe |

**关键决策**：
1. 遇到 Rust 不可用时，果断切换到 Electron 方案
2. 网络问题导致 Electron 下载失败时，配置国内镜像解决
3. 代码签名步骤失败时，使用 `build:dir` 直接输出未签名版本

**最终产物**：`dist/win-unpacked/markdown-editor-reader.exe`
